stm32f1xx_hal_dma库的使用

本文最后更新于：2023年8月7日 下午

下面来看看 DMA HAL库关于 DMA 使用方法。

DMA 外设库的使用

  • 使能和配置外设连接到 DMA通道。 (除了内部 SRAM / FLASH 内存: 不用初始化。) 注意： 在内存到内存的传输模式中，不允许循环模式 。请阅读参考手册关于外设和DMA请求的内容。
    1. 对于给定的通道，配置以下参数 HAL_DMA_Init()：
      • 通道请求 Channel request
      • 传输方向 Transfer Direction
      • 源数据和目标数据的格式 Source and Destination data formats
      • 循环或正常模式 Circular or Normal mode
      • 通道优先级水平 Channel Priority level
      • 源数据和目标数据增量模式 Source and Destination Increment mode
    2. 获取DMA状态，如果出错，获取错误信息。HAL_DMA_GetState()和HAL_DMA_GetError()
    3. 中止当前传输。HAL_DMA_Abort()
  • 轮询模式下IO的操作
    • 配置完源地址和目标地址以及数据长度后，开启DMA数据传输。HAL_DMA_Start()
    • 传输完当前数据，用户可以在程序中添加超时时间。HAL_DMA_PollForTransfer()
  • 中断模式下IO的操作
    • 配置DMA中断优先级。HAL_NVIC_SetPriority()
    • 使能DMA IRQ中断。HAL_NVIC_EnableIRQ()
    • 配置完源地址和目标地址以及数据长度后，开启DMA中断模式下的数据传输。 HAL_DMA_Start_IT()
    • 调用DMA中断函数。HAL_DMA_IRQHandler() ，DMA_IRQHandler()
    • 数据传输完成，执行回调函数。HAL_DMA_IRQHandler() ，XferCpltCallback ，XferErrorCallback ...
  • DMA HAL库常用宏定义列表
    • __HAL_DMA_ENABLE: 使能指定的 DMA 通道。
    • __HAL_DMA_DISABLE: 关闭指定的 DMA 通道。
    • __HAL_DMA_GET_FLAG: 获取DMA通道挂起标志。
    • __HAL_DMA_CLEAR_FLAG: 清除DMA通道挂起标志。
    • __HAL_DMA_ENABLE_IT: 使能指定DMA通道中断。
    • __HAL_DMA_DISABLE_IT: 关闭指定DMA通道中断。
    • __HAL_DMA_GET_IT_SOURCE: 检查指定的DMA通道中断是否发生。
