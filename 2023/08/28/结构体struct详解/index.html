结构体struct详解

本文最后更新于：2023年8月30日 晚上

结构体(struct)是 C 编程中一种用户自定义的数据结构，它可以包含不同基本类型(char,int,float...)的数据项，在结构体中的变量，可以是相同、部分相同，或完全不同的数据类型。需要注意的是，结构体是创建变量的模板，不占用内存空间；结构体变量才包含了实实在在的数据，会占用内存空间。

在实际应用中，我们一般会使用结构体将某个对象的属性和行为进行封装声明，然后定义一个独立的整体变量进行使用。

结构体的声明与定义

结构体的声明和定义有多种形式：

/* 1.声明一个结构体然后进行定义 */
struct tag
{
    int a;
    char b;
    double c;
};//分号不能少
struct tag t1, t2[10], *t3; //定义了三个struct变量

/* 2.声明一个结构体的同时进行定义 */
struct tag
{
    int a;
    char b;
    double c;
}t1, t2;

/* 3.声明一个结构体的同时进行定义,同时去掉tag标签 */
struct 
{
    int a;
    char b;
    double c;
}t1, t2; 

/* 4.用typedef创建新类型，比较常用 */
typedef struct
{
    int a;
    char b;
    double c;
}data_t;
data_t t1, t2[10], *t3; //定义了三个struct变量

结构体变量的初始化

和其它类型变量一样，对结构体变量可以在定义时指定初始值。用符号{}括起来，用逗号分隔各个数据项。

struct tag
{
  int a;
  char b;
  double c;
};//分号不能少
struct tag t1 = {
  20,
  "libs",
  890.234
}; //各个数据项要与声明时数据项一一对应

在C99标准中，还支持给指定元素赋值：

struct tag t1 = {
  .c = 890.234
  .a = 20,
  .b = "libs",
}; //各个数据项可以跟声明时数据项不对应

结构体成员变量的访问

如果定义的是结构体变量指针，我们需要用->来访问结构体的成员；如果定义的是结构体变量，我们需要用.来访问结构体成员。

#include<stdio.h>

struct Date
{
	int year;
	int month;
	int day;
};
struct Student
{
	char s_name[20];
	struct Date birthday;
	float score;
};
int main()
{
	struct Student stu = { "liuhai",2000,10,10,99.5 };
	struct Student *p_stu; //定义一个结构体指针
	p_stu = &stu;

	printf("name=%s\n birtyday=%d.%d.%d\n score=%f\n", stu.s_name, stu.birthday.year, stu.birthday.month, stu.birthday.day, stu.score);

	printf("name=%s\n birtyday=%d.%d.%d\n score=%f\n", p_stu->s_name, p_stu->birthday.year, p_stu->birthday.month, p_stu->birthday.day, p_stu->score);
	
	return 0;
}

函数与结构体参数

结构体和结构体指针都可以作为函数参数来使用。如下：

#include<stdio.h>

typedef struct school
{
	char s_name[20];
	int s_age;
}school_t;

/* 结构体作为函数参数 */
void print_a(school_t sx)
{
	printf("name: %s age: %d\n", sx.s_name, sx.s_age);
}
/* 结构体指针作为函数参数 */
void print_b(struct school* sp)
{
	printf("name: %s age: %d\n", sp->s_name, sp->s_age);
}
int main()
{
	struct school sx = { "wuhan", 20 };
  print_a(sx);
	print_b(&sx);
	return 0;
}

在实际应用中，因为结构体作为行参在调用中是复制一份结构体实参，占用内存比较大，而结构体指针大小都一样，占用内存就不会太大，所以，使用结构体指针进行函数参数传递的方式更好一些。

结构体大小的计算与对齐

不同的CPU架构，各种基础数据类型的大小不一样，具体如下：

关于结构体大小的计算，需要把握三个规则：

  • 结构体变量的首地址，必须是结构体变量的“最大基本数据类型成员所占字节数”的整数倍；

  • 结构体变量中的每个成员相对于结构体首地址的偏移量，都是该成员基本数据类型所占字节数的整数倍；

  • 结构体变量的总大小，为结构体变量中“最大基本数据类型成员所占字节”的整数倍；

CPU并非逐字节读写内存，而是以2,4,或8的倍数的字节块来读写内存，因此会对基本数据类型的地址进行字节对齐排列。程序中预处理指令 # pragma pack(n) 可以改变默认对齐数。n取值有 1,2,4,8,16 。
