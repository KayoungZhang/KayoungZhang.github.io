stm32f1xx_hal_gpio库的使用

本文最后更新于：2023年8月8日 早上

怎么使用stm32库文件呢？在 *.c文件中, ST官方已经给出了使用方法，下面具体看看GPIO外设库中的特点和使用步骤：

GPIO外设库的特点

  1. GPIO外设库可独立配置模式：

    • 输入模式 Input mode

    • 模拟模式 Analog mode

    • 输出模式 Output mode

    • 复用功能模式 Alternate function mode

    • 外部中断或事件模式 External interrupt/event lines

  2. 复位期间和之后，复用功能和外部中断线没有被激活，I/O端口被配置为输入浮空模式。

  3. 所有的GPIO引脚内部都有弱上拉和下拉电阻，可选择激活或不激活。

  4. 在输出或复用模式，每一个I/O端口可配置为开漏或推挽模式，根据VDD电压值可选择I/O速度。

  5. 所有的端口都有外部中断和事件的功能，当使用外部中断线时，端口必须配置为输入模式，所有的可用GPIO引脚连接到16个外部中断、事件线（EXTI0 ~ EXTI15）。

  6. 对于互联型MCU，外部中断、事件控制器包含多达20个边沿检测器，其它类型MCU，也有19个边沿检测器，这些检测器用于产生中断、事件请求。每条输入线都能进行独立配置选择类型（事件或中断），相应的触发条件（上升沿、下降沿、上升下降沿）。每条线也可以独立屏蔽，挂起寄存器来维持中断请求线的状态。

GPIO外设库的使用步骤

  1. 使能GPIO APB2时钟。__HAL_RCC_GPIOx_CLK_ENABLE()

  2. 配置GPIO引脚。HAL_GPIO_Init()

    • 配置IO端口模式。GPIO_InitTypeDef结构体中Mode成员。

    • 激活上下拉电阻。GPIO_InitTypeDef结构体中Pull成员。

    • 当输出或复用模式时，配置I/O速度。GPIO_InitTypeDef结构体中Speed成员。

    • 当引脚用于ADC或DAC时，配置IO端口模式为模拟模式。

    • 当外部中断、事件时，配置IO端口模式为中断或事件模式，同时触发条件选择上升沿或下降沿或上升下降沿。

  3. 当外部中断、事件时，配置NVIC IRQ映射到EXTI线的优先级，并使能。HAL_NVIC_SetPriority()，HAL_NVIC_EnableIRQ()

  4. 获取在输入模式下引脚的电平。HAL_GPIO_ReadPin()

  5. 设置或复位输出模式下引脚的电平。HAL_GPIO_WritePin()/HAL_GPIO_TogglePin()

  6. 锁定引脚配置直到下一次复位。HAL_GPIO_LockPin()

  7. 复位期间和之后，复用功能未激活，GPIO引脚被配置为输入浮空模式（JTAG引脚除外）。

  8. 当LSE振荡器关闭时，其引脚OSC32_IN/OSC32_OUT(PC14和PC15)可作为IO口使用，LSE的优先级高于GPIO的功能。

  9. 当HSE振荡器关闭时，其引脚OSC_IN/OSC_OUT(PD0和PD1)可作为IO口使用，HSE的优先级高于GPIO的功能。
