C语言程序数据存储结构

本文最后更新于：2023年8月18日 下午

一般内存主要分为：代码区、常量区、静态区（全局区）、堆区、栈区这五个区域：

keil 环境下默认的内存配置如下图：

C程序在编译链接后形成的映像（image）中，将只包含 代码段(Text)、只读数据段(RO Data) 和 读写数据段(RW Data) 。在程序运行之前，将动态生成 未初始化数据段(BSS段)，在程序的运行时还将动态生成 堆(Heap)区域 和 栈(Stack)区域 。

下面结合STM32内核和代码实际测试各存储区及其特点：

代码区（code segment/text segment）

存放可执行程序的代码，即CPU执行的机器指令，由系统控制，并且是只读的。

  • 代码区的地址:函数的地址,程序的入口地址，函数的名称也是一个指针，可以通过查询函数名称所处的内存地址，查询函数存放的区域。

  • 字符串常量和define定义的常量也有可能存放在代码区。

void test(void)
{
  printf("main: #%p\r\n", main);//打印main函数的存放地址
}

可以看到main函数确实存放在0x08002e81这片地址区域，在代码区中。

常量区（rodata segment）

存放常量，程序运行的期间，常量区内容只可访问，不可修改。

  • 字符串、数字等常量存放在常量区。
  • const修饰的全局变量存放在常量区。
void test(void)
{   
  char *p="abcdef"; //定义指针p指向字符串abcdef
  char str[] = "abcd"; //定义字符数组str

  printf("0x%p: %s\r\n", p , *p);//打印指针p的值和p指向的字符串内容
  printf("0x%p: %s\r\n", str , *str);//打印字符数组的地址和字符串内容
  p="qwedma";                       //将指针p指向字符串qwedma
  printf("0x%p: %s\r\n", p , *p);//打印指针p的值和p指向的字符串内容
  p[0]='1';                     //尝试把p指向的第一个字符q修改为1
  str[0]='e';                   //尝试把字符数组第一个字符a修改为e
  printf("0x%p: %s\r\n", p , *p);//打印指针p的值和p指向的字符串内容
  printf("0x%p: %s\r\n", str , *str);  //打印字符数组的地址和字符串内容
}

可以看到，abcdef字符串常量存放在0x08002ecc区域，p指针指向该区域，并把p指针移走指向qwedma字符串常量的首地址0x08002ee0。当尝试修改p指向的第一个字符，即把qwedma修改为1wedma，发现打印出来的内容并未改变，常量区的内容不可以改变。

可以看到str是指向栈区的地址：0x200007c0，且指向的内容可以被修改，第一个字符a被修改为e。

全局(静态)区

用于存储在整个程序运行期间都可见的全局变量和static修饰的变量。全局(静态)区对应三个段： 未初始化数据段(BSS Data)，读写数据段(RW Data)，只读数据段(RO Data) 。

.bss段（bss segment）Block Started by Symbol

  • 存储未初始化的或初始化为0的全局变量和静态变量
  • 不占用可执行文件空间，其内容由操作系统初始化。

.data段（data segment）

  • 存储已初始化且不为0的全局变量和静态变量。
  • 编译期间其大小及数据被确定，占用可执行文件空间，其内容由程序初始化。
static int a; //a是静态全局变量
int c; //c是全局变量
void test(void)
{
    static int b=1; //b是静态局部变量
    b++;
    printf("b:0x%p: %d\n", &b , b);
}
int main()
{
    printf("a: 0x%p: %d\n", &a , a);
    printf("c: 0x%p: %d\n", &c , c);
    for(uint8_t i=0;i<5;i++)
    {
        test();
    }
    return 0;
}

可以看到它们都存放在静态区，编译器自动把未初始化的a和c初始化为0；b在for循环中初始化5次，且b的地址一直是不变的，说明编译器只会为第一次初始化的b分配内存，后面4次初始化是无效的。

堆区

  • 堆区由程序员分别调用malloc()和free()函数分配和释放内存。若申请了堆区内存，之后忘记释放内存，很容易造成内存泄漏
  • 堆区按内存地址由低到高方向生长，其大小由系统内存/虚拟内存上限决定，速度较慢，但自由性大，可用空间大。
void test(void)
{   
  int *p1 = (int*)malloc(sizeof(int));//申请4字节的空间
  int *p2 = (int*)malloc(sizeof(int));//申请4字节的空间
  printf("p1:0x%p\n",p1); //打印申请的空间首地址
  printf("p2:0x%p\n",p2); //打印申请的空间首地址
  *p1=123;// 该空间赋值123
  printf("0x%p:%d\n",p1,*p1); //打印指针p1的值和p1指向的值
  printf("0x%p\n",&p1); //打印p1的地址
  free(p1); //释放p1指向的值
  printf("0x%p:%d\n",p1,*p1); //打印p1的值和p1指向的值
  p1 = NULL;   //将p1指针释放，避免成为野指针
  printf("0x%p\n",p1);
}

可以看到申请到的空间首地址为0x200003e8，存在是堆区； 在释放空间时，同时应该释放相应的指针(置为NULL)。

p2的地址是大于p1的地址，验证堆区是向上生长的，后申请的空间地址会依次增加。

栈区

  • 临时创建的局部变量和const定义的局部变量及函数形参、返回值存放在栈上。
  • 栈区按内存地址由高到低方向生长，先进后出原则。
  • 栈区由编译器自动分配释放，无须手动管理。栈上内容在函数运行结束就会自动销毁。
void test(void)
{   
    int a;
    int b=0;    
    printf("a:0x%p:%d\n",&a,a);
    printf("b:0x%p:%d\n",&b,b);
}

可以看到后定义的局部变量b地址是比a小的，即栈区是向下生长的; a变量没有进行初始化，打印出的a是垃圾值，编译器不会把局部变量初始化为0。但是，需要注意：如果你运行于debug调试模式，运行时机制会把你的栈空间全部初始化为0，这就是解释了为什么平时在debug时看到的未初始化的局部变量初始值是0.所以使用局部变量时，尽量要先进行初始化，避免垃圾值造成错乱。
