算法中的时间和空间复杂度

本文最后更新于：2023年7月7日 下午

算法简介

所谓算法（Algorithm），即解决问题的方法。同一个问题，使用不同的算法，虽然得到的结果相同，但耗费的时间和资源肯定有所差异。

注意，程序和算法是完全不同的概念。算法是解决某个问题的想法、思路；而程序是在根据算法编写出来的真正可以运行的代码。例如，要依次输出一维数组中的数据元素的值，首先想到的是使用循环结构，在这个算法的基础上，我们才开始编写程序。

程序的运行效率具体可以从 2 个方面衡量，分别为：

  • 程序的运行时间。（时间维度）
  • 程序运行所需内存空间的大小。（空间维度）

程序运行时间越短，运行期间占用的内存越少，该程序运行效率就越高，这样算法也就更好。衡量一个算法所编写出程序的运行效率在数据结构中，用时间复杂度来衡量程序运行时间的多少；用空间复杂度来衡量程序运行所需内存空间的大小。

算法中复杂度的表示方法

在数据结构中，每条语句的执行次数，被称为该语句的频度，整段代码的总执行次数，为整段代码的频度。所以算法复杂度的一般通用表示方法是：「 大O符号表示法 」，即 O(频度)。大O符号表示法是一种算法复杂度的相对表示方式。

时间复杂度

时间复杂度概念

一个算法所编程序运行时间的多少，并不是在计算机上运行所消耗的时间来度量的，而是根据合理方法得到的预估值。那么，如何预估一个算法所编程序的运行时间呢？就是计算程序中每行代码的执行次数，然后用总的执行次数间接表示程序的运行时间，即：

T(n) = O(f(n))

其中：T(n)表示程序执行的时间；f(n) 表示每行代码执行次数之和；而 O 表示 T(n) 与 f(n) 成正比关系，这个公式的全称是：算法的渐进时间复杂度。

我们先来看个例子：

for(int i = 0 ; i < n ; i++)     //<- 从 0 到 n，执行 n+1 次
{
    a++;                         //<- 从 0 到 n-1，执行 n 次
}

因此，整段代码中所有语句共执行了 (n+1)+n 次，即 2n+1 次。

再看一个例子：

for(int i = 0 ; i < n ; i++)           // n+1次
{ 
    for(int j = 0 ; j < m ; j++)       // n*(m+1)次
    {
        num++;                         // n*m次
    }
}

这段代码一共执行的次数是：(2*n*m+2*n+1)。当 n、m 都无限大时，可认为 n==m，则可以简化为 2n²+2n+1，这就是此段程序在最坏情况下的运行频度。

在数据结构中，一般会使用 无限大的思想 来简化频度表达式。可以这样简化：

  • 首先去掉常数分子。例如 2n²+2n+1 简化为 2n²+2n ；
  • 然后有多项无限大的变量的，只保留一个最高项变量,且最高项系数为1。例如 2n²+2n 简化为 n²；

这样，就可以得到以上两个例子的时间复杂度分别为 O(n) 和 O(n²) 。

常见的时间复杂度量级

常见的时间复杂度量级有：

  • 常数阶O(1)
  • 对数阶O(logn)
  • 线性阶O(n)
  • 线性对数阶O(nlogn)
  • 平方阶O(n²)
  • 立方阶O(n³)
  • 指数阶(2^n)

上面从上至下依次的时间复杂度越来越大，执行的效率越来越低。注意，这里仅以最坏情况下的频度作为时间复杂度，而在某些实际场景中，还可以用最好情况下的频度和最坏情况下的频度的平均值来作为算法的平均时间复杂度。

下面分别举例说明：

  1. 常数阶O(1)

无论代码执行了多少行，哪怕执行上万行，只要是没有循环等复杂结构，那这个代码的时间复杂度就都是O(1),如：

int i = 1;
int j = 2;
++i;
j++;
int m = i + j;
  1. 对数阶O(logn)
int i = 1;     //执行1次
while(i<n)     //执行log2^n次
{
    i = i * 2; //执行log2^n次
}

这段代码总的执行次数是 2log2^n + 1; 简化：log2^n, 对数的底是不确定的，这里恰好是2，一般会去掉对数的底，所以可以进一步简化：logn，即时间复杂度：O(logn) 。

  1. 线性阶O(n)

如：

for(i=1; i<=n; ++i) //执行了n次
{
   j++ = i;   //执行了 n-1 次
}

这段代码执行了 2n-1 次，简化后，即时间复杂度 O(n) 。

  1. 线性对数阶O(nlogn)

将时间复杂度为O(logn)的代码循环n遍的话，那么它的时间复杂度就是 n * O(logn)，也就是了O(nlogn)。

for(m=1; m<n; m++) //执行n次
{
    i = 1;       //执行n次
    while(i<n)   //执行 n*log2^n次
    {
        i = i * 2;  //执行n*log2^n次
    }
}

这段代码执行了 2n+2nlog2^n 次，简化后，即时间复杂度 O(nlogn) 。

  1. 平方阶O(n²)

如果把 O(n) 的代码再嵌套循环一遍，它的时间复杂度就是 O(n²) 了。

for(x=1; i<=n; x++) //执行n次
{
   for(i=1; i<=n; i++) //执行n*n次
    {
       j++ = i;  //执行n*n次
    }
}

这段代码执行了 n+2n² 次，简化后，即时间复杂度 O(n²) 。

  1. 立方阶O(n³)

O(n³)相当于三层n循环，其它的类似。

  1. 指数阶(2^n)

在递归调用程序中，会有指数阶，如：

int f(int n)
{
   if (n == 0) return 1;
   return f(n-1) + f(n - 1);
}

这里执行的次数计算: 2^0 + 2^1 + 2^2 + …… + 2^n，即2^(n+1)-1,进一步简化则时间复杂度:O(2^n) 。

除此之外，其实还有 平均时间复杂度、均摊时间复杂度、最坏时间复杂度、最好时间复杂度 ，有点复杂，这里这里就不展开了。

空间复杂度

空间复杂度是对一个算法在运行过程中临时占用存储空间大小的一个量度，同样反映的是一个趋势，我们用 S(n) 来定义。和时间复杂度类似，也用 「 大O符号表示法 」 来表示。

事实上，对算法的空间复杂度影响最大的，往往是程序运行过程中所申请的临时存储空间。如：

int n;
scanf("%d", &n);
int a[10];

这段程序在运行时所申请的临时空间，并不随 n 的值而变化。而如果将第 3 行代码改为int a[n], 此时程序运行所申请的临时空间，和 n 值有直接的关联。

所以，如果程序所占用的存储空间和输入值无关，则该程序的空间复杂度就为 O(1)；反之则需要进一步判断它们之间的关系：

  • 输入值 n 不断增大，程序申请的临时空间成线性增长，则程序的空间复杂度用 O(n) 表示;
  • 输入值 n 不断增大，程序申请的临时空间成 n² 关系增长，则程序的空间复杂度用 O(n²) 表示；
  • 输入值 n 不断增大，程序申请的临时空间成 n³ 关系增长，则程序的空间复杂度用 O(n³) 表示；

在多数场景中，一个好的算法往往更注重的是时间复杂度的比较，而空间复杂度只要在一个合理的范围内就可以。空间复杂度比较常用的有：O(1)、O(n)、O(n²)
