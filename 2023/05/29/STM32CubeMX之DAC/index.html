STM32CubeMX之DAC

本文最后更新于：2023年6月2日 早上

DAC（digital to analog converter）即数模转换器，它可以将数字信号转换为模拟信号.在常见的数字信号系统中，大部分传感器信号被转化成电压信号，而ADC把电压模拟信号转换成易于计算机存储、处理的数字编码，由计算机处理完成后，再由DAC输出电压模拟信号来驱动某些执行器件，使人类易于感知。如音频信号的采集和还原就是这样的一个过程。

STM32F1的DAC模块是12位电压输出型DAC。可以配置为8位或12位模式，在12位模式下，数据可以设置为左对齐或右对齐。它有2个输出通道，2个通道可以独立地进行转换。

硬件设计

本实验通过 DAC 通道1 输出模拟电压，用KEY1用来增加DAC输入值，WK_UP用来减少DAC输入值，然后使用USART1将输出的电压值打印出来

软件设计

STM32CubeMX设置

  • RCC设置外接HSE，时钟设置为72M
  • PE3设置为GPIO输入模式、上拉；PA0设置为GPIO输入模式，下拉；

  • USART1选择为异步通讯方式，波特率设置为115200Bits/s，传输数据长度为8Bit，无奇偶校验，1位停止位
  • 激活DAC_OUT1，关闭输出缓冲，不使用触发功能

  • 输入工程名，选择工程路径（不要有中文），选择MDK-ARM V5；勾选Generate peripheral initialization as a pair of ‘.c/.h’ files per peripheral；点击GENERATE CODE，生成工程代码

MDK-ARM编程

  • 因为使用到按键功能，所以移植之前用到的按键输入工程中Key文件,记得添加头文件和路径（#include "key.h"）。

  • 在uart.c文件中添加fputc重定义函数，并添加头文件<stdio.h>，勾选keil软件Use MicroLIB选项。
  • 在main.c文件下编写DAC测试代码
/* USER CODE BEGIN 1 */
uint16_t dac_val;		//读取的DAC值
float dac_vol;		//转换后的电压值
uint16_t dac_setval = 0;	//DAC设置值
uint8_t key = 0;
/* USER CODE END 1 */

/* USER CODE BEGIN 2 */
HAL_DAC_Start(&hdac,DAC_CHANNEL_1);	//开启DAC通道1
/* USER CODE END 2 */

while (1)
{
    key = key_scan(0);

    if(key == KEY1_PRES)
    {
        if(dac_setval > 4000)
            dac_setval = 0;
        dac_setval += 200;
    }
    else if(key == WKUP_PRES)
    {
        if(dac_setval < 200)
            dac_setval = 4000;
        dac_setval -= 200;
    }

    HAL_DAC_SetValue(&hdac,DAC_CHANNEL_1,DAC_ALIGN_12B_R,dac_setval);

    if(key == KEY1_PRES || key == WKUP_PRES)
    {
        dac_val = HAL_DAC_GetValue(&hdac,DAC_CHANNEL_1);//其值=dac_setval
        printf("DAC_OUT1 DAC value: %d\r\n",dac_val);
        dac_vol = (float)dac_val * (3.3/4096);
        printf("DAC_OUT1 VOL value: %.2fV\r\n",dac_vol);
    }
    HAL_Delay(10);
}
  • 编译无误后下载到开发板，打印出DAC通道1的DAC值和电压值，按下按键，DAC值和电压值发生变化。
