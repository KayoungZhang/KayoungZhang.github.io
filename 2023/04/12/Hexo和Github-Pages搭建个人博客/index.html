Hexo和Github Pages搭建个人博客

本文最后更新于：2023年5月4日 晚上

1. 前言

目前网上有很多第三方博客类型，像博客园，CSDN，简书，知乎等。也可以自己搭建个人技术博客，这里以Hexo+Github Pages为例搭建个人博客。Hexo是一个简单、快速、强大的基于 Node.js的静态博客发布工具，支持Markdown格式，有众多优秀插件和主题。Github Pages 是 Github 提供的一种免费的静态网页托管服务，可以用来托管博客、项目官网等静态网页。搭建步骤如下：

2. 开发环境

需要安装开发环境Node.js和git

安装完成后，在桌面右键选择 Git Bash Here ，命令窗口打开后，分别输入git --version, node -v, npm -v, 如果弹出对应的版本号，则说明安装正确，否则，需要查找原因。

3. 安装 Hexo

在你的电脑上创建一个博客文件夹，如：F:\Blog, 进入此文件夹，右键打开Git Bash Here,依次输入

npm install hexo-cli -g  # 安装hexo
hexo init                # 初始化，安装所需包
npm install              # 新版本的Hexo在初始化时就安装好了依赖包
hexo g                   # 生成
hexo s                   # 启动本地服务

打开浏览器访问 http://localhost:4000 即可看到博客网页内容。第一次初始化的时候hexo已经帮我们写了一篇名为 Hello World 的文章。

4. 配置SSH key

ssh key用来解决本地和服务器的连接访问github权限问题。

4.1 本地生成密钥对

在git bash命令行中执行 cd ~/.ssh, 如果弹出 No such file or directory,说明你是第一次使用git。需要生成ssh key,否则说明你已经配置了SSH key，不需要再次操作。 在命令行中执行 ：

git config --global user.name "填入你的github用户名"
git config --global user.email "填入你的email地址"
ssh-keygen -t rsa -C "填入你的email地址"

连续三次回车，最终会在用户目录下生成文件，在用户目录下（C:）找到.ssh\id_rsa.pub文件，用记事本打开并复制里面的所有内容。

4.2 添加公钥到Github

打开你的github主页，进入Settings -> SSH and GPG keys -> New SSH key：

添加完成后，还需要确认并添加主机到本机SSH可信列表，在 git bash中输入 ssh -T git@github.com ,若返回 Hi xxx! You've successfully authenticated, but GitHub does not provide shell access.则证明添加成功。

5. 创建github远程仓库，并与本地blog仓库关联

进入你的github主页，创建一个新的仓库，因为Github 仅能使用一个同名仓库的代码托管一个静态站点，所以Repository名字必须是你的Github名.github.io，比如我的是KayoungZhang.github.io

打开本地blog仓库中配置文件 **_config.yml**，在末尾添加：

deploy:
	type: git
	repository: git@github.com:你的用户名/你的用户名.github.io.git
	#上一步的Github仓库地址，项目主页点SSH再复制url，不要采用http形式的url
	branch: main
	#填写远程仓库的分支名，新版本默认是main分支，老版本是master分支

在blog文件夹中打开git bash命令行，安装hexo 部署插件，然后进行部署。

npm install hexo-deployer-git --save
hexo d

在浏览器中输入网址： 你的用户名.github.io ，即可查看到网页。一般来说如果出现莫名的问题，按照以下步骤来解决。

删除本地blog仓库中.deploy_git文件夹
git bash中执行命令 
hexo clean
hexo g
hexo d

6. 写博客

在本地blog仓库中，打开git bash，执行命令 hexo new 'my-first-blog'(my-first-blog是文件名)，hexo会帮我们在source/_posts下生成my-first-blog.md文件，打开文件，hexo自动添加了一些说明的文件头，后面只需要自己添加文章内容即可。写完后，执行以下命令完成网页上传部署，一般情况下编辑文章中的图片在网页上会无法正常显示，需要图床，将图片上传到网上，同时在主题目录下head.jade文件中添加<meta name="referrer" content="no-referrer" />，具体操作和原因，参看博客 PicGo + Gitee(码云)实现markdown图床，Hexo使用细节及各种问题

hexo clean
hexo g -d

7. 个性化你的博客

7.1 更换主题

默认主题不好看，可以进入 hexo主题官网 选择自己喜欢的主题，进行更改。这里以 beautiful-hexo 主题为例。进入本地blog文件夹中，右键打开 Git Bash Here, 根据选择主题的 README.md文件执行相应命令:

npm install --save hexo-generator-archive hexo-renderer-jade hexo-generator-tag hexo-generator-feed hexo-generator-sitemap hexo-browsersync 

git clone --depth 1 https://github.com/twoyao/beautiful-hexo.git themes/beautiful-hexo

在blog文件夹中找到配置文件 **_config.yml** ,使用记事本打开，找到 theme: landscape , 将 landscape改为 beautiful-hexo,保存后，执行命令 hexo clean, 清理一下public的内容，然后执行 hexo g来重新生成, 最后打开本地服务器，执行 hexo s ,打开浏览器就会看到新的主题界面。

7.2 主题配置

打开主题目录下的配置文件 **_config.yml**，可以进行自修改，不使用的功能进行注释（语句前添加#）

header:
   title: "Beautiful Hexo"
   motto: 不积跬步无以至千里，不积小流无以成江海
   bigimgs:
       - src: /bigimgs/01.jpg
         desc: beautiful island
       - src: /bigimgs/02.jpg
         desc: Tropical Fish
       - src: /bigimgs/03.jpg
         desc: ocean
mail:
  link: 'mailto:zky0830@163.com'
  title: Email me
  icon: fa-envelope
  enable: true
wechat:
  link: '/wechat.png' #将图片放到source目录下
  title: Wechat
  icon: fa-wechat
  enable: true

7.3 全局配置

打开配置文件 _config.yml，修改相应内容：

# Site
title: My Tech Blog #主页标题
subtitle: '' #副标题
description: '' #网站描述
keywords:
author: KayoungZhang #作者
language: zh-cn #可根据themes主题中language选择相应语言
timezone: ''

# URL
## Set your site url here. For example, if you use GitHub Page, set url as 'https://username.github.io/project'
url: https://KayoungZhang.github.io #项目的 GitHub Pages网址
permalink: :year/:month/:day/:title/
permalink_defaults:
pretty_urls:
  trailing_index: true # Set to false to remove trailing 'index.html' from permalinks
  trailing_html: true # Set to false to remove trailing '.html' from permalinks

#自定义站点内容搜索，需要安装插件：
# npm install hexo-generator-search --save
search:
  path: search.xml
  field: all # 如果只索引文章，可设置为post

7.4 创建文档、标签和分类

创建标签和分类是为了便于管理和查找，为了给文章自动添加标识，比较方便的做法是，打开scaffolds目录下的post.md文档，添加tags: 和 categories:

---
title: {{ title }}
date: {{ date }}
tags:
categories:	
---

这样以后创建文档，它就会自动在文档开头添加标识，你只需要在相应标识下添加分类和标签（以-+空格开头）即可，注意：hexo 一篇文章只能属于一个分类，不会产生两个分类，而是把分类嵌套。如下：

title: Hexo和Github Pages搭建个人博客
date: 2023-04-12 18:57:46
tags:
 - hexo
 - 入门
categories: 
 - Blog

进入本地blog仓库，打开git bash，通过不同的命令，创建相应的内容。

7.4.1 创建文章
hexo new "填入你的文章名"

在source/_posts 目录下就会生成创建的文档，打开进行写作即可。

7.4.2 创建标签
hexo new page tags

在source/tags/目录下会生成index.md文档，打开此文档，添加 layout: "tags"到文档中保存退出，如：

---
title: tags
date: 2023-04-12 11:12:33
layout: "tags"
---

要想在主页显示标签栏，需要进入主题配置文件 _config.yml，添加标签栏

navbar-links:
	...
	Tags: /tags

点击首页的 “tags” 可以看到该标签页下的所有文章。当然，只有文章中添加了tag属性的才会被收录到首页的 “标签” 中。

7.4.3 创建分类
hexo new page categories

在source/categories目录下生成的index.md文档，打开文档，添加 layout: "categories"到内容中,保存退出，如：

---
title: categories
date: 2023-04-12 19:05:47
layout: "categories"
---

同理，要想在主页显示分类栏，需要进入主题配置文件 _config.yml，添加分类栏：

navbar-links:
  ...
  Categories: /categories

对于beautiful-hexo主题，还需要在主题的layout目录下创建categories.jade文件，内容如下（因为不会前端知识，所以根据tags.jade编写）：

extends partial/base

block header
    header.header-section
        .intro-header.no-img

block content
    .container
        .row
            .col-lg-6.col-lg-offset-3.col-md-10.col-md-offset-1

                - site.categories.map(function(category) {  
                    if category.posts.length > 0
                        ul.archive-list
                            span.fa.fa-category.listing-seperator(id=category.name, style="font-size: 1.17em;")
                            span(style="font-size: 1.5em; font-weight: bold; margin-left: 5px;")= category.name
                            - category.posts.map(function(post) {
                                li.col-lg-offset-1.col-md-offset-1
                                    span.post-meta= full_date(post.date, 'MMM D')
                                    a.post-title(href=url_for(post.path))= post.title
                            - })
                - })

点击首页的 “categories” 可以看到该分类下的所有文章。 参考博客：使用Hexo创建文章、标签和分类

8. Hexo博客同时发布到github和gitee

因为github服务器在国外，所以打开网页比较慢，国内的gitee pages具有同样的部署功能，打开网页就会更快一些，所以可以将hexo博客同步到gitee上，它的部署发布与github差不多，就是每次更新完博客后，gitee部署时需要进入Pages页面进行手动更新，不再过多赘述，具体参照这篇博客：hexo博客同时发布到github和gitee

本地博客文件同时上传到github和gitee：如何将一个项目同时提交到GitHub和Gitee(码云)两个仓库

9. 多台设备同步更新Hexo博客

真正的原始Hexo博客内容在我们的电脑本地，上传到网页上的内容是public文件夹里面的静态内容，有的时候写文章需要在不同的电脑上操作，这样就需要博客项目在不同设备间的迁移与同步，具体操作参照博客： Hexo在多台电脑上提交和更新 多台电脑同步更新Hexo博客

以上是备份本地博客内容到云端仓库，所以所有人都能看到你的源文件，如果不想让人看到你的源文件，只需在云端仓库新建一个仓库，再与本地博客仓库关联起来即可。之后在不同的设备上，git clone这个云端新仓库，然后按照正常方式操作即可。
