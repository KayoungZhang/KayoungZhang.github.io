stc单片机免断电自动下载程序方法

本文最后更新于：2023年4月17日 早上

学过stc单片机的人，应该都知道，下载程序时，需要先加载程序，然后断电再上电，方可将程序下载到单片机中运行。有的开发板做了硬件电路处理，结合下载软件的设置，就不用断电也可以进行程序下载，下面介绍一下另外一种方式，硬件电路不需要如何修改，只通过软件设置就可以进行免断电下载程序。

在 STC 官方下载软件（STC-ISP）中，点击下载按钮后， 其实从电脑端串口是一直在发送数据的，一般是 0x7F，也可以是用户自定义， 根据STC12C5A60S2芯片手册软件复位一章中，设置 IAP_CONTR寄存器（STC89是ISP_CONTR寄存器） 就可以实现不断电复位，从而进入ISP下载区进行程序的下载。

第一种方式：利用串口下载程序瞬间一直发送 0x7f

/* 方案一：串行口中断,用于stc单片机不断电自动下载程序 */
uint8_t uCount = 0; //计数
void UART1_Handler(void) interrupt 4
{
	ES = 0; //关闭串口中断
	if(RI)
	{
		RI = 0; //Clear receive interrupt flag
		if(SBUF == 0x7F)
		{
            uCount++;
            if(uCount > 9)   //isp软件用户自定义命令连续发送10个0x7F，即可实现自动热复位下载
            {
                uCount = 0;
                IAP_CONTR |= 0x60;    //软复位到系统ISP监控区
            }
		}
		else
		{
			uCount = 0; //计数清零
		}
	}
	ES = 1; //重新打开串口中断。
}

第二种方式：利用stc-isp下载工具默认内部自定义命令”@STCISP#“

void UART1_Handler(void) interrupt 4
{
	char* str = "@STCISP#";//stc internal definition command
	ES = 0; //关闭串口中断
	if(RI)
	{
		RI = 0; //Clear receive interrupt flag
		while(*str != '\0')
		{
			if(*str++ == SBUF)
				continue;
			IAP_CONTR |= 0x60;
		}
	}
	else
		TI = 0;
	ES = 1; //重新打开串口中断。
}

选择以上任何一种方式，保证 USB-CDC/串口模式 中波特率跟自己程序中串口1的波特率一致，在main函数的开头初始化一下串口1，就可以实现不断电下载程序了。
