hexo博客更换主题详细方法

本文最后更新于：2023年8月28日 下午

在创建hexo个人博客时，已经简单介绍了更换主题的方法，下面主要是介绍我在更换 fluid 主题后，进行的一些自定义设置及碰到的问题和解决方法。

1. 下载 fluid 主题及修改配置文件

在本地 Hexo 博客目录中打开 git bash，然后下载 fluid 主题。

git clone https://github.com/fluid-dev/hexo-theme-fluid.git themes/fluid

修改 Hexo 博客目录中的配置文件 **_config.yml** :

theme: fluid
language: zh-CN #根据主题目录下language文件夹选择相应语言

2. 创建「关于页」

fluid主题默认已经创建了首页，分类，归档，标签等项，但主题的「关于页」需要手动创建:

hexo new page about

创建成功后，编辑博客目录下 /source/about/index.md，添加 layout 属性。修改后的文件示例如下：

---
title: about
date: 2023-04-16 10:16:49
layout: about
---

#这里写关于页的正文，添加你需要在关于页展示的内容，支持 Markdown, HTML

此时，运行以下命令，打开 http://localhost:4000/,即可查看到你的本地网页博客。

hexo clean
hexo g
hexo s

3. 自定义设置

打开 fluid 主题目录下配置文件 **_config.yml** ，里面详细介绍了各种设置的方法，并进行了中文翻译。

3.1 更换 页面光标闪烁

# Cursor character
cursorChar: "✍"

3.2 修改导航栏博客名

blog_title: "My Tech Blog"

3.3 添加页脚 个人版权信息

content: '
    <p <i class="iconfont icon-copyright"></i> <span>2023 | KayougZhang</span> </p>
    <a href="https://hexo.io" target="_blank" rel="nofollow noopener"><span>Hexo</span></a>
    <i class="iconfont icon-love"></i>
    <a href="https://github.com/fluid-dev/hexo-theme-fluid" target="_blank" rel="nofollow noopener"><span>Fluid</span></a>
  '

3.4 修改首页显示的图像

找到自己喜欢的 png 格式图片，命名为 default.png ，然后复制到主题目录 source/img 下，进行替换原图片。

3.5 修改post页面显示的图像

找到自己喜欢的 png 格式图片，命名为 article.png ，然后复制到主题目录 source/img 下。

post:
  banner_img: /img/article.png
  banner_img_height: 70
  banner_mask_alpha: 0.3

3.6 修改首页宣传语

slogan:
  enable: true

  # 为空则按 hexo config.subtitle 显示
  # If empty, text based on `subtitle` in hexo config
  text: "积跬步以至千里，积小流而成江海"

3.7 文章开头显示更新时间

updated:
  enable: true

3.8 关闭文章目录显示和版权声明

toc:
  enable: false

copyright:
  enable: false

3.9 设置评论插件

打开评论插件需要第三方服务，所以需要进行第三方设置，根据fluid评论配置指南，选择 Twikoo，进行 Zeabur 部署，进行它的提示进行设置，最后会得到一个 envId, 此处我设置的是 https://ENV, 在配置文件进行如下设置：

comments:
  enable: true
    # 指定的插件，需要同时设置对应插件的必要参数
    # The specified plugin needs to set the necessary parameters at the same time
    # Options: utterances | disqus | gitalk | valine | waline | changyan | livere | remark42 | twikoo | cusdis | giscus
    type: twikoo 
    
# Twikoo
# 基于腾讯云开发 , Zeabur 部署
# Based on Tencent CloudBase
# See: https://twikoo.js.org
twikoo:
  envId: https://ENV
  region: ap-shanghai
  path: window.location.pathname

如果选用valine，需要进入LeanCloud，申请APP ID 和 APP Key，然后同上面一样在配置文件中进行如下设置：

comments:
  enable: true
    # 指定的插件，需要同时设置对应插件的必要参数
    # The specified plugin needs to set the necessary parameters at the same time
    # Options: utterances | disqus | gitalk | valine | waline | changyan | livere | remark42 | twikoo | cusdis | giscus
    type: valine
    
# Valine
# 基于 LeanCloud
# Based on LeanCloud
# See: https://valine.js.org/
valine:
  appId: '填入申请的appID'
  appKey: '填入申请的appKey'
  path: window.location.pathname
  placeholder: "欢迎留言交流"
  avatar: 'robohash'
  meta: ['nick', 'mail', 'link']
  requiredFields: []
  pageSize: 10
  lang: 'zh-CN'
  highlight: false
  recordIP: true
  serverURLs: ''
  emojiCDN:
  emojiMaps:
  enableQQ: false

3.10 更改「about」页图标

在 about 配置栏中，删除名字和介绍的部分，在 图标用法中，选择需要添加的图标，然后进行设置，我添加了 github，mail 和 微信，所以，需要准备我的微信图片，命名为 wechat.png ,复制到主题目录 source/img 下。

about:
  enable: true
  banner_img: /img/default.png
  banner_img_height: 60
  banner_mask_alpha: 0.3
  avatar: /img/avatar.png
  name: ''
  intro: '' 
icons:
  - { class: "iconfont icon-github-fill", link: "https://github.com/KayoungZhang", tip: "GitHub" }
  - { class: "iconfont icon-mail", link: "mailto:zky0830@163.com", tip: "Email me" }
  - { class: "iconfont icon-wechat-fill", qrcode: "/img/wechat.png" }

3.11 更改「about」页avatar图像

avatar图像可进行修改，也可以进行删除，修改方式是，找到自己喜欢的图片，命名为 avatar.png , 粘贴到 img 文件夹中进行替换，此处我直接删除，操作方式如下：

打开主题目录下 layout/layout.ejs 文件, 删除以下代码：

<% if(page.layout === 'about') { %>
  <div class="about-avatar">
    <img src="<%= url_for(theme.about.avatar) %>" class="img-fluid" alt="avatar">
  </div>
<% } %>

3.12 调整 post 页面的大小

在post页中，文章内容部分页面很窄，有的文章排版成挤缩状，看起不舒服，进行如下设置：

打开主题目录下 layout/post.ejs 文件, 将 col-lg-8 修改为 12：

<div class="col-lg-12 nopadding-x-md">

以上是我采用 fluid 主题的一些详细设置，因为不会 javascript，所以，后面几项设置折腾了两天，现在记录一下。

3.13 更改代码显示的样式

打开 fluid 主题目录下配置文件 **_config.yml** ，修改：

highlightjs:
    # 在链接中挑选 style 填入
    # Select a style in the link
    # See: https://highlightjs.org/static/demo/
    # style: "github gist"
    style: "Vs"
    style_dark: "dark"

4. 添加数学公式支持

打开主题目录下的配置文件 _config.yml,将markdown公式支持打开。

math:
  # 开启后文章默认可用，自定义页面如需使用，需在 Front-matter 中指定 `math: true`
  # If you want to use math on the custom page, you need to set `math: true` in Front-matter
  enable: true

  # 开启后，只有在文章 Front-matter 里指定 `math: true` 才会在文章页启动公式转换，以便在页面不包含公式时提高加载速度
  # If true, only set `math: true` in Front-matter will enable math, to load faster when the page does not contain math
  specific: true

  # Options: mathjax | katex
  engine: mathjax

下载安装： pandoc，然后进入本地博客目录下，右键打开git bash,安装 hexo-renderer-pandoc

npm uninstall hexo-renderer-marked --save  
npm install hexo-renderer-pandoc --save

根据 \node_modules\hexo-renderer-pandoc目录下的 README.md 文档，打开本地博客主目录下的配置文件： _config.yml，添加pandoc软件的路径。

pandoc:
  pandoc_path: C:/Program Files/Pandoc/pandoc.exe

注意： 有时部署博客文章会出现关于pandoc的错误，可根据提示，进行相应的修改。

在需要显示数学公式的文档前面添加 math: true,之后页面就会显示公式。为了方便，可以打开 \scaffolds\post.md文件，添加math:,这样，在以后需要显示公式的文档中将math后面添加true即可。
