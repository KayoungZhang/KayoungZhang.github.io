git的使用教程

本文最后更新于：2023年8月28日 下午

git 是世界上最先进的开源分布式版本管理工具，相比于svn集中式版本管理工具，git可以让开发人员更加方便的查看和操作项目。

1. git安装

软件下载：git官网 或 国内镜像

2. 配置本地git标识(用户名和邮箱)

打开 git bash,输入：

git config --global user.name "John Doe"
git config --global user.email johndoe@example.com

John Doe 替换成你的用户名，johndoe@example.com 替换成你的邮箱。设置这两个信息的原因是在于远程库进行关联时需要一个key，配置这两个信息以确保你的唯一性。

使用--global参数表明你这台机器上所有的仓库都进行相同的配置。

3. git的基本操作

git init  # 初始化git仓库，会在当前目录生成一个.git隐藏文件夹，不要去修改这个文件夹下的任意东西。
git add [file] #添加文件到暂存区
git commit -m "填写提交说明信息"   #提交暂存区中的内容到本地仓库
git log   # 查看提交日志，可查看到版本commit ID
git status # 查看所有文件的状态
git reset --hard HEAD^   #上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写成HEAD~100
git reset --hard commit id #通过commit ID回退到那个版本，ID只需要前几位即可

4. git忽视文件

有些文件是不想被git管理可以通过配置从而达到忽视掉一些文件，这样这些文件就可以不用提交了。在仓库的根目录创建一个.gitignore的文件,然后将不需要被管理的文件添加到此文件中即可。如：

log.txt  # 忽视log.txt文件
.gitignore  # 忽视.gitignore文件
css/*.js  # 忽视css下的所有的js文件
css/*.*  # 忽视css下的所有文件

5. Git分支与合并

git branch                      #查看本地分支
git branch [branch-name]        #新建一个分支，但依然停留在当前分支
git checkout -b [branch]        #新建一个分支，并切换到该branch分支
git checkout [branch-name]      #切换到指定分支，并更新工作区
git branch -d [branch-name]     #删除本地分支
git merge [branch]              #合并指定branch分支到当前分支

6. 关联远程仓库

远程仓库主要有：github和gitee。

gitee服务器在国内，网速比较快，github服务器在国外，访问速度比较慢，但用户数比较多。

  • 进入他们的官网，按照要求自行注册一个账户。

  • 远程仓库绑定本地仓库，就可以SSH免密登陆，在git bash命令行窗口输入ssh-keygen-t rsa ,一直默认按回车即可，在本地目录：C:\Users\Administrator\.ssh 文件中会生产两个秘钥文件，用记事本打开公钥文件 id_rsa.pub，复制里面的内容。

  • 在 github 网页上，依次打开 settings -> SSH and GPG keys页面，新创建SSH key，填上任意Title，在Key文本框里粘贴刚才复制的内容，添加即可。（gitee类似操作）

  • 在 github 网页上新建一个远程仓库，然后实现云端仓库和本地仓库同步。（gitee类似操作）

git remote add origin [远端仓库地址]   #把本地仓库与远程仓库进行关联
git push -u origin [本地分支名字]  #推送分支的所有内容到远程仓库

7. 将远程仓库内容复制到本地

git clone [url]  　#获取github上的远程仓库
git pull [url]     #拉取github上远程仓库变化的内容

8. git的一些常用命令

  • 新建代码库
git init         # 在当前目录新建一个Git代码库
git clone [url]  #clone 远程仓库到本地
  • 本地仓库更改
git add [file1] [file2] ...    # 添加指定文件到暂存区
git add .     					# 添加当前目录的所有文件到暂存区
git diff       					# 显示暂存区和工作区的差异
git rm [file1] [file2] ...     # 删除工作区文件，并且将这次删除放入暂存区
git commit -m [message]        # 提交暂存区到仓库区

git status                  # 显示有变更的文件
git log                      # 显示当前分支的版本历史

  • 分支与标签 

    git branch       # 列出所有本地分支
git branch -r    # 列出所有远程分支
git branch [branch-name]    # 新建一个分支，但依然停留在当前分支
git checkout -b [branch]   # 新建一个分支，并切换到该分支
git checkout [branch-name]  # 切换到指定分支，并更新工作区
git branch -d [branch-name]  # 删除分支
git merge [branch]  # 合并指定分支到当前分支

git tag [tag-name] # Mark the current commit with a tag
git tag -d [tag]   # 删除本地tag
git show [tag]  # 查看tag信息

  • 更新与发行

git fetch [remote]  # 下载远程仓库的所有变动
git remote -v   # 显示所有远程仓库
git remote show [remote]   # 显示某个远程仓库的信息
git remote add [shortname] [url]  # 增加一个新的远程仓库，并命名
git pull [remote] [branch]   # 取回远程仓库的变化，并与本地分支合并
git push [remote] [branch]   # 上传本地指定分支到远程仓库
git push [remote] --force  # 强行推送当前分支到远程仓库，即使有冲突

git merge <branch>   #Merge <branch> into your current HEAD
git rebase <branch>  #Rebase your current HEAD onto <branch>
  • 撤销
git checkout [file]  # 恢复暂存区的指定文件到工作区
git reset [file]   # 重置暂存区的指定文件，与上一次commit保持一致，但工作区不变
git reset --hard   # 重置暂存区与工作区，与上一次commit保持一致
git reset [commit]   # 重置当前分支的指针为指定commit，同时重置暂存区，但工作区不变
git reset --hard [commit]   # 重置当前分支的HEAD为指定commit
git reset --keep [commit]  # 重置当前HEAD为指定commit，但保持暂存区和工作区不变
$ git revert [commit]  # 新建一个commit，用来撤销指定commit
